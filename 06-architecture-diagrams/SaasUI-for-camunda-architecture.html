④ Configurable Persistence Router
🗄
Platform Storage Mode
domain_object_configs: mode=platform
  • Writes JSONB to step_submissions
  • platform_data column
  • Default for zero-infra tenants
  • Atomic per domain object
🌐
External API Mode
domain_object_configs: mode=external_api
  • HTTP POST to tenant-configured URL
  • Auth: Bearer / API key / Basic / OAuth2
  • Field-ID → request body field mapping
  • Retry + exp. backoff + dead-letter queue
  • Correlation-ID idempotency header
💾
External DB Mode
domain_object_configs: mode=external_db
  • PostgreSQL · MySQL · MSSQL · Oracle
  • Encrypted connection string at rest
  • Column mapping per field ID
  • Per-tenant isolated connection pool
  • No cross-tenant pool reuse
⚠ Non-blocking rule:  External persistence failures NEVER block Camunda task completion — the workflow advances and failed domain objects are queued for async retry. Partial failure is reported per domain object. Each can be retried independently without re-submitting successful ones.
